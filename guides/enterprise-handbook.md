## 2. Gateway 多平台接入

Gateway 是 Hermes 的消息入口，让你的 AI Agent 可以接收来自不同平台的消息并统一处理。

### 已配置平台

| 平台 | 状态 | 用途 |
|------|------|------|
| **[平台1]** | ✅ 已配置 | 日常团队沟通 |
| **[平台2]** | ✅ 已配置 | 客户通知 |
| **[平台3]** | ✅ 已配置 | 自动化工作流 |

### 使用方式

**团队成员在对应平台 @Agent 即可：**

```
@AgentX 今天的竞品动态汇总
@AgentX 帮我查一下 React 19 的发布说明
@AgentX 提醒我下午 3 点开会
```

Agent 会在同一平台回复，不需要打开终端。

### 安全策略（allowlist）

当前只允许以下用户/群组与 Agent 交互：

```
# Telegram
allowlist:
  users: [@xxx, @xxx]
  groups: [@xxx_team]

# 飞书
allowlist:
  users: [xxx, xxx]
  chats: [xxx_tech_group]
```

> ⚠️ 如需添加新成员，联系我修改 allowlist。这是为了防止未授权用户滥用 API token。

### 添加新平台

支持对接的平台列表：
- Telegram / Discord / Slack（海外）
- 飞书 / 钉钉 / 企业微信（国内）
- 自定义 Webhook

需要新增平台时联系我，配置时间约 30 分钟/平台。

## 3. Cron 定时任务

定时任务是 Hermes 的自动化核心。设定好之后，Agent 会在指定时间自动执行。

### 你的定时任务列表

| # | 任务名 | 执行时间 | 内容 |
|---|--------|----------|------|
| 1 | **[任务1]** | 每天 9:00 | [描述] |
| 2 | **[任务2]** | 每天 18:00 | [描述] |
| 3 | **[任务3]** | 每周一 10:00 | [描述] |
| 4 | **[任务4]** | [时间] | [描述] |
| 5 | **[任务5]** | [时间] | [描述] |

### 管理定时任务

**查看所有任务：**
```bash
hermes cronjob list
```

**查看某条任务的执行日志：**
```bash
hermes cronjob logs --name "任务名"
```

**手动触发一次（测试用）：**
```bash
hermes cronjob trigger --name "任务名"
```

**暂停/恢复：**
```bash
hermes cronjob pause --name "任务名"
hermes cronjob resume --name "任务名"
```

### 常见 Cron 表达式

| 场景 | 表达式 |
|------|--------|
| 每天 9:00 | `0 9 * * *` |
| 每天 18:00 | `0 18 * * *` |
| 每周一 10:00 | `0 10 * * 1` |
| 每 30 分钟 | `*/30 * * * *` |
| 每月 1 号 9:00 | `0 9 1 * *` |

### 创建新任务示例

```bash
# 每日竞品监测
hermes cronjob create \
  --name "每日竞品监测" \
  --cron "57 8 * * *" \
  --prompt "搜索 [竞品名] 和 [竞品名] 的最新动态，总结三条要点，发到团队群"

# 每周开发周报
hermes cronjob create \
  --name "每周开发周报" \
  --cron "3 18 * * 5" \
  --prompt "分析本周 git 提交记录，生成一份开发周报，包括：完成的功能、修复的 bug、下周计划"
```

### 技能清单

| 技能名 | 触发方式 | 功能 |
|--------|----------|------|
| **[技能1]** | "帮我做 XXX" | [描述] |
| **[技能2]** | "帮我做 XXX" | [描述] |
| **[技能3]** | "帮我做 XXX" | [描述] |

### 技能文件位置

```bash
~/.hermes/skills/custom/
├── [skill-1]/
│   └── skill.md
├── [skill-2]/
│   └── skill.md
└── [skill-3]/
    └── skill.md
```

### 技能结构

每个技能文件包含：
- **name** — 技能名称
- **description** — 一句话描述（Hermes 用这个判断何时调用）
- **instructions** — 详细执行指令
- **examples** — 示例输入输出

### 修改技能

技能是纯文本 Markdown 文件，直接编辑即可：

```bash
nano ~/.hermes/skills/custom/[skill-name]/skill.md
```

修改后无需重启，下次调用时自动生效。

### 新增技能

如果需要新的业务技能，告诉我需求，我来开发。也可以自己写——参考现有技能文件的格式。

## 6. 安全加固

### 已实施的安全措施

- [x] API Key 权限最小化（DeepSeek Key 仅限 API 调用）
- [x] Gateway allowlist（只允许指定用户/群）
- [x] 敏感文件权限设置（`.env` 仅当前用户可读）
- [x] Tirith 安全扫描已配置
- [x] 命令执行确认（高危操作需二次确认）

### 日常安全检查

**每月做一次：**
1. 检查 API 用量：登录 DeepSeek 后台 → 查看本月消耗
2. 检查 Gateway 访问日志：
   ```bash
   hermes gateway logs --since "30 days ago" | grep ERROR
   ```
3. 更新 Hermes：
   ```bash
   hermes update
   ```

### API Key 泄露应急流程

1. 立即登录 DeepSeek 后台 → 删除泄露的 Key
2. 生成新 Key
3. 更新 `~/.hermes/.env` 里的 Key
4. 重启 Hermes
5. 联系我确认安全

### 团队成员离职

1. 从 Gateway allowlist 移除该成员
2. 如果该成员接触过 API Key，轮换所有 Key
3. 检查最近的执行日志是否有异常操作

## 7. 团队使用指南

### 角色和权限

| 角色 | 能做什么 | 不能做什么 |
|------|----------|------------|
| **管理员**（你） | 全部操作、修改配置、管理成员 | - |
| **开发者** | 通过平台 @Agent 提问、使用定时任务输出 | 修改 Agent 配置 |
| **查看者** | 接收定时任务推送 | 直接与 Agent 交互 |

### 团队接入步骤

1. 成员在对应平台（飞书/Telegram 等）加入团队群
2. 管理员将成员 ID 加入 allowlist
3. 成员发送 `@AgentX 你好` 测试
4. 收到回复即接入成功

### 最佳实践

**提问技巧：**
- 具体比模糊好：「帮我查一下上周的销售数据」> 「查数据」
- 给上下文：「我们用的是 Next.js 14，App Router 模式，想加一个...」
- 追问别怕：「再详细点」「给我看看代码示例」

**团队规范建议：**
- 复杂任务让一个人统一与 Agent 对话，避免多人同时下指令造成混淆
- 定时任务输出建议统一发到专用群，不要混在业务群里
- API 用量超预算时，优先停掉非关键定时任务

## 8. 运维与监控

### 日常检查清单

**每天（自动化）：**
- Cron 任务是否正常执行（查看推送）

**每周（手动 5 分钟）：**
```bash
hermes cronjob list          # 确认所有任务状态正常
hermes gateway logs --since "7 days ago" | tail -20  # 快速扫一眼异常
```

**每月（手动 10 分钟）：**
1. 登录 DeepSeek 后台查用量
2. 检查磁盘空间（对话日志会累积）
3. 备份自定义技能文件
4. 评估是否需要调整定时任务

### 备份

自定义文件备份建议：

```bash
# 备份到云盘或 Git
tar -czf agentx-backup-$(date +%Y%m%d).tar.gz \
  ~/.hermes/skills/custom/ \
  ~/.hermes/.env \
  ~/.hermes/config.yaml \
  ~/.hermes/USER.md \
  ~/.claude/CLAUDE.md
```

### 升级流程

1. 备份当前配置（见上面）
2. 更新 Hermes：`hermes update`
3. 更新 Claude Code：`claude --update`
4. 测试：各发一条消息确认正常
