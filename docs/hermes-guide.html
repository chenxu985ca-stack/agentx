Hermes Agent 安装指南

覆盖 macOS / Windows / Linux 三大平台
从零到跑通第一条消息，跟着做就行

🍎 macOS 🪟 Windows 🐧 Linux
Before You Start

前置准备

💡
Hermes Agent 的安装脚本会自动安装 Python、Node.js、ripgrep 等依赖，你几乎不需要手动装任何东西。唯一需要提前准备的是 Git 和 API Key。
1

安装 Git（如未安装）

在终端里运行 git --version，如果显示版本号就说明已经有了，跳过此步。

没有的话：macOS 运行 xcode-select --install；Windows 去 git-scm.com 下载；Linux 用 sudo apt install git 或对应包管理器安装。

2

准备 API Key（推荐 DeepSeek）

去 platform.deepseek.com 注册账号，在 API Keys 页面创建一个 Key 并复制保存。新用户通常有免费额度。

其他可选：OpenAI、Anthropic (Claude)、OpenRouter、Kimi (Moonshot) 等。Hermes 支持 20+ 模型提供商。

💰
DeepSeek 是目前性价比最高的选择，日常使用月均 ¥10-50。V4 Pro 模型还支持 128K 上下文。
3

国内用户：网络准备

如果访问 GitHub 很慢或超时，安装命令里用 ghfast.top 镜像加速即可（下面的命令已经帮你替换好了）。

Choose Your Platform

安装步骤

选择你的操作系统，按步骤操作

1

安装 Xcode Command Line Tools

打开终端（Terminal.app），运行：

bash 
xcode-select --install

如果已安装会提示 "already installed"，不影响后续步骤。

2

一键安装 Hermes

在终端中运行以下命令（复制整行，粘贴到终端，回车）：

bash 
curl -fsSL https://ghfast.top/https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

脚本会自动检测你的系统、安装缺失的依赖（Python、Node.js、ripgrep、ffmpeg），然后把 hermes 装到 ~/.local/bin/hermes。

🍺
也可以用 pip 安装：pip install hermes-agent && hermes postinstall（需要先有 Python 3.10+）
3

刷新终端配置

安装完成后，让终端识别新的 hermes 命令：

bash 
source ~/.zshrc

macOS 默认用 zsh。如果你用的是 bash，改为 source ~/.bashrc。或者直接关掉终端重新打开也行。

4

验证安装

运行版本检查：

bash 
hermes --version

预期输出类似 hermes v0.14.0。再来个全面体检：

bash 
hermes doctor

看到 Model: connected 就说明一切就绪。

💡
Windows 有两套方案：
方案 A（推荐）：WSL2 — 最稳定，所有功能正常工作
方案 B：原生 PowerShell — 仍在 beta，Dashboard 终端功能受限
如果你不确定选哪个，选方案 A。

方案 A：WSL2（推荐）

1

启用 WSL2

以管理员身份打开 PowerShell，运行：

powershell 
wsl --install -d Ubuntu

安装完成后会提示重启电脑。重启后会自动打开 Ubuntu 终端，设置用户名和密码（牢记）。

⚠️
需要 Windows 10 版本 2004+ 或 Windows 11。不支持 WSL1，必须是 WSL2。
2

在 WSL2 里安装 Hermes

打开 Ubuntu 终端（开始菜单搜 "Ubuntu"），运行：

bash 
curl -fsSL https://ghfast.top/https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
bash 
source ~/.bashrc
3

验证

bash 
hermes --version
hermes doctor

方案 B：原生 PowerShell（Beta）

🧪
此方案为早期测试版，Dashboard 的 /chat 交互终端不支持（需 POSIX PTY）。CLI、Gateway、Cron、MCP Server 等功能正常。如果遇到问题，请换方案 A。
1

运行安装脚本

打开 PowerShell（不需要管理员权限），运行：

powershell 
irm https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.ps1 | iex

脚本会自动安装 Python 3.11、Node.js 22、ripgrep、ffmpeg 和 PortableGit（如尚未安装）。

2

重开终端，验证

关掉 PowerShell，重新打开，然后：

powershell 
hermes --version
hermes doctor
1

安装基础依赖

大多数发行版已经自带 curl 和 git。如果没有：

bash 
# Debian / Ubuntu
sudo apt update && sudo apt install curl git -y

# Fedora
sudo dnf install curl git -y

# Arch
sudo pacman -S curl git
2

一键安装 Hermes

bash 
curl -fsSL https://ghfast.top/https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

脚本会自动安装 Python、Node.js、ripgrep、ffmpeg 等依赖。全程不需要 sudo。

3

刷新终端配置

bash 
source ~/.bashrc

如果你用的是 zsh：source ~/.zshrc。或者关掉终端重新打开。

4

验证安装

bash 
hermes --version
hermes doctor

如果 hermes: command not found，确认 ~/.local/bin 在 PATH 里：

bash 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
Configuration

安装后配置

安装完成 ≠ 能正常使用，还需要配好模型和身份

1

配置模型提供商

编辑 ~/.hermes/config.yaml，把 model 部分改成：

yaml ~/.hermes/config.yaml 
model:
  default: deepseek-v4-pro
  provider: deepseek
  base_url: https://api.deepseek.com/v1

如果你用其他模型（Claude、OpenAI 等），把 provider 和 base_url 改成对应的即可。运行 hermes model 可以交互式配置。

2

设置 API Key

编辑 ~/.hermes/.env，填入你的 API Key：

bash ~/.hermes/.env 
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
🔐
.env 文件权限应设为 600：chmod 600 ~/.hermes/.env。Hermes 会在日志中自动屏蔽 API Key，但仍建议妥善保管。
3

定制 Agent 身份

用自然语言告诉 Hermes 它是谁、你的习惯是什么：

markdown ~/.hermes/memories/USER.md 
# About Me

- 我是独立开发者，主要用 TypeScript 和 Python
- 工作日 9:00-18:00 在线
- 回复语言偏好中文

USER.md 告诉 Hermes 关于你的事，MEMORY.md 是它自己积累的记忆。两个文件都在 ~/.hermes/memories/ 下。

4

安装推荐技能

Hermes 有 90+ 社区技能，推荐装这几个最常用的：

bash 
hermes skills install research
hermes skills install github
hermes skills install productivity
hermes skills install software-development

装完后 hermes skills list 查看所有已安装技能。

5

首次启动

一切就绪，启动吧：

bash 
hermes

第一次启动会自动进入设置向导（也可以手动 hermes setup）。看到聊天提示符就说明安装成功了。试试说 "你好"。

Troubleshooting

常见问题

GitHub 访问慢 / curl 下载失败怎么办？▾
上面教程里的安装命令已经用了 ghfast.top 镜像加速。如果还是不行，可以试试：
1. 开代理/科学上网
2. 用 pip 方式安装：pip install hermes-agent && hermes postinstall（配置 PyPI 镜像：pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple）
安装完输入 hermes 提示 command not found？▾
终端没有刷新 PATH。运行 source ~/.zshrc（macOS）或 source ~/.bashrc（Linux/WSL），或者直接关掉终端重开。
如果还不行，手动加：echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
API Key 配置后报 401 / 模型不回复？▾
常见原因：
1. API Key 写错了（多了空格、引号）— 去平台后台重新复制
2. 模型名不对 — DeepSeek 用 deepseek-v4-pro，不是 deepseek-chat
3. 环境变量名不对 — DeepSeek 是 DEEPSEEK_API_KEY，不是 OPENAI_API_KEY
4. 账号没余额 — 去平台后台确认
macOS 提示 "Permission denied"？▾
可能是之前用了 sudo 安装导致文件权限异常。清理：sudo rm /usr/local/bin/hermes，然后重新安装（不要加 sudo）。Hermes 设计为用户级安装，全程不需要管理员权限。
Windows WSL2 里 hermes 找不到？▾
关掉 WSL 终端，PowerShell 里运行 wsl --shutdown，等几秒后重新打开 Ubuntu。这会让 WSL 完全重载环境变量。
后续使用还需要付费吗？▾
Hermes Agent 本身免费开源。你只需要付 API 调用费：DeepSeek 约 ¥1-3 / 百万 token，日常使用月均几十块钱。相比 ChatGPT Plus 月费 $20 便宜很多。
