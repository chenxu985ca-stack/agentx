💻 Claude Code 完整指南

适用于 macOS / Windows / Linux · 从安装到精通

1. IDE 选择与环境准备

部署 Claude Code 前，推荐安装一个 Agent IDE（免费即可）：

IDE下载特点
Cursorcursor.com最流行，AI 原生集成
Traetrae.ai（国内）/ trae.ai/en（海外）字节出品，国内可用
Google Anti-gravityidx.google.comGoogle 出品，云端 IDE
以下教程以 macOS 为主，Windows 差异处已标注。

2. 安装 Claude Code

macOS

步骤 1：装 Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
步骤 2：把 Homebrew 加到 PATH

装完后终端会有提示，或者直接让 Claude Code 帮你配：

帮我把 homebrew 加到 PATH 路径变量里面去
步骤 3：安装 Claude Code
brew install --cask claude-code@latest
步骤 4：验证
claude --version

Windows

步骤 1：安装 Git（遇到选项输入 y）
winget install Git.Git
步骤 2：安装 Claude Code

PowerShell（IDE 终端默认）：

irm https://claude.ai/install.ps1 | iex

CMD：

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
步骤 3：把 Claude Code 加进 PATH
winget install Anthropic.ClaudeCode
步骤 4：重启 IDE 终端后验证
claude --version

3. 配置大模型

方案 A：DeepSeek API（推荐，国内友好）

# 写入环境变量
cat >> ~/.zshrc << 'EOF'
export ANTHROPIC_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
export ANTHROPIC_BASE_URL=https://api.deepseek.com/v1
EOF

source ~/.zshrc

Windows 用户：在系统环境变量里添加这两条。

方案 B：Anthropic API（官方）

export ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

方案 C：CC Switch 多模型管理

支持多种模型自由切换的工具。

安装：

  1. 下载：CC Switch v3.14.1
  2. 根据系统选择安装包
  3. 务必在启动 Claude Code 之前先设置好 CC Switch

三种工作模式：

默认模式增强版对话终端，用于意图确认和信息查询
计划模式（Plan）分析需求、制定步骤，不直接动代码
Accept Edits执行代码变更的"落地"阶段

4. 权限模式

Claude Code 在 Accept Edits 模式下执行命令时会询问：

选项含义
仅同意这一次只允许当前这次命令执行
同意且不再询问该项目之后执行同类操作不再问
不同意拒绝，再商量

如果希望一路绿灯自动执行：

claude --dangerously-skip-permissions
这会跳过所有权限确认，建议只在信任的项目中使用。

6. CLAUDE.md 配置

CLAUDE.md 是 Claude Code 的项目规则文件，启动时自动读取。

全局配置（所有项目生效）

nano ~/.claude/CLAUDE.md
# CLAUDE.md

## About Me

- 角色：全栈独立开发者
- 技术栈：Next.js / TypeScript / PostgreSQL
- 偏好：代码注释用英文，commit message 用英文
- 回复用中文

## Work Style

- 修改代码前先读文件
- 保持与周围代码风格一致
- 优先使用项目已有的工具库

项目级配置

cd /path/to/your/project
nano CLAUDE.md
# CLAUDE.md

## Project Info

- Next.js 14 App Router
- TypeScript strict mode
- pnpm as package manager
- Testing: Vitest + Playwright

## Rules

- Use server components by default
- API routes in app/api/, follow RESTful
- Run tests before committing

项目级 vs 全局级

全局 CLAUDE.md项目 CLAUDE.md
位置~/.claude/CLAUDE.md项目根目录
作用范围所有项目仅当前项目
优先级低高（会覆盖全局）
适合放什么个人信息、通用风格项目技术栈、特定规则

8. 子 Agent（SubAgent）

子 Agent 是 Claude Code 可调用的独立执行单元，适合并行处理多任务。

创建子 Agent

  1. 输入 /agents，在 Library 下选择创建项目级子 Agent
  2. 描述用途（如"专门处理数据库迁移"）
  3. 选择模型（一般选 Sonnet；用了 CC Switch 则选哪个都一样）
  4. 为子 Agent 选一个区别于主 Agent 的颜色

调用与管理

输入 /agents，在 Library 下选择已创建的子 Agent。

适合：并行代码审查、多文件重构、独立模块开发。

15. 双 Agent 协同配置

如果同时使用 Hermes，在 CLAUDE.md 中加入：

## Co-Agent: Hermes

This workspace has Hermes Agent installed as a co-agent.

### Division of Labor

| Claude Code | Hermes |
|---|---|
| Code writing, editing, refactoring | Web research & info gathering |
| Git operations, commits, PRs | Scheduled/cron tasks |
| Debugging, testing, code review | Multi-platform messaging |
| File system operations | Long-running workflows |

### When to Suggest Hermes

- 搜索类任务 → 建议用 Hermes
- 定时/周期任务 → Hermes cronjob
- 多平台消息 → Hermes gateway

17. 常见问题

问题排查
command not found: claudenpm 全局路径不在 PATH；Windows 检查环境变量
401 UnauthorizedAPI Key 不对或 ANTHROPIC_BASE_URL 配错
走 DeepSeek 模型名报错Claude Code 默认调 Claude 模型，需映射
权限被拒工作目录需读写权限
启动慢首次启动下载依赖，等 1-2 分钟
