🤖 双 Agent 协同使用手册

你现在拥有一个双人 AI 团队：Claude Code 负责写代码，Hermes 负责搜资料和自动化。这份手册教你如何让它们高效配合。

1. 系统概览

你的 AI 团队架构：

你（人类） ├── Claude Code（写代码、改文件、Git 操作） └── Hermes（查资料、定时任务、多平台消息）
Claude CodeHermes
定位你的 AI 程序员你的 AI 助理
擅长写代码、重构、Debug、Git搜索、定时任务、跨平台
运行方式终端 CLI，按需启动常驻后台，随时可用
启动命令claudehermes

2. Claude Code 快速上手

启动

cd /path/to/your/project
claude

基本用法

直接描述你想做的事，Claude Code 会自己读代码、写代码、执行命令：

> 帮我修复这个登录页面的 bug
> 给这个函数加单元测试
> 重构一下这个模块，拆成两个文件

常用操作

操作做法
写新功能直接描述需求，Claude Code 会自己规划并实现
修 Bug描述 bug 现象或粘贴报错信息
代码审查「帮我 review 一下这次的改动」
提交代码「帮我 commit 这些改动」
询问代码「这个函数是干嘛的？」

关键概念：

  • CLAUDE.md — 项目规则文件，Claude Code 启动时自动读取。已包含双 Agent 分工规则
  • Memory — Claude Code 会记住你的偏好和项目信息
  • 权限确认 — 执行修改前会征求你的同意

5. 协同工作流实战

场景一：开发新功能

第 1 步：Hermes 做调研（10min）

> 帮我搜一下 Next.js App Router 的最佳实践，总结 5 条要点
> 查一下 XXX 库的最新 API 文档

第 2 步：Claude Code 写代码（30min）

> 根据 Hermes 搜到的最佳实践，帮我实现一个新路由 /dashboard
> 参考这个 API 文档，对接 XXX 接口

第 3 步：Claude Code 写测试 + 提交

> 给刚才的代码加测试
> commit 一下，message 写 "feat: add dashboard route"

场景二：修 Bug

第 1 步：Claude Code 定位问题

> 用户反馈登录后白屏，帮我排查一下

Claude Code 会自己读报错日志、追踪代码、定位根因。

第 2 步：（可选）Hermes 搜索类似问题

> 搜一下 "Next.js middleware redirect infinite loop" 社区怎么解决

第 3 步：Claude Code 修复 + 验证

> 按这个方案修一下，然后跑一下登录流程的测试

场景三：每日工作流

早上：

Hermes:      帮我总结昨晚 GitHub 上关注项目的更新
Claude Code: （继续昨天的开发任务）

开发中：

  • Claude Code 写代码
  • 遇到不确定的技术选型 → Hermes 搜索对比

下班前：

Claude Code: 帮我总结今天改了什么，生成 commit message
Hermes:      把这些改动摘要发到团队 Telegram

7. 常见问题

Q: 两个 Agent 会冲突吗？

不会。它们运行在不同进程中，读写不同的配置文件。

Q: 可以同时开两个吗？

可以。两个终端窗口，左边 Claude Code 右边 Hermes，各干各的。

Q: API 费用怎么算？

两个 Agent 共用同一个 DeepSeek API Key。建议在 DeepSeek 后台设置月度预算上限。

Q: 国内 GitHub 访问慢？

已配置镜像加速。如果还有问题，联系我。
