Agent 会在同一平台回复，不需要打开终端。

安全策略（allowlist）

当前只允许以下用户/群组与 Agent 交互：

# Telegram
allowlist:
  users: [@xxx, @xxx]
  groups: [@xxx_team]

# 飞书
allowlist:
  users: [xxx, xxx]
  chats: [xxx_tech_group]
如需添加新成员，联系我修改 allowlist。这是为了防止未授权用户滥用 API token。

可扩展平台

3. Cron 定时任务

定时任务是 Hermes 的自动化核心——设定后 Agent 按时自动执行。

你的定时任务

#任务名执行时间内容
1[任务1]每天 9:00[描述]
2[任务2]每天 18:00[描述]
3[任务3]每周一 10:00[描述]
4[任务4][时间][描述]
5[任务5][时间][描述]

管理命令

hermes cronjob list                      # 查看所有任务
hermes cronjob logs --name "任务名"       # 查看执行日志
hermes cronjob trigger --name "任务名"    # 手动触发（测试用）
hermes cronjob pause --name "任务名"      # 暂停
hermes cronjob resume --name "任务名"     # 恢复

Cron 表达式速查

场景表达式
每天 9:000 9 * * *
每天 18:000 18 * * *
每周一 10:000 10 * * 1
每 30 分钟*/30 * * * *
每月 1 号 9:000 9 1 * *

创建新任务示例

# 每日竞品监测
hermes cronjob create \
  --name "每日竞品监测" \
  --cron "57 8 * * *" \
  --prompt "搜索 [竞品名] 的最新动态，总结三条要点，发到团队群"

# 每周开发周报
hermes cronjob create \
  --name "每周开发周报" \
  --cron "3 18 * * 5" \
  --prompt "分析本周 git 提交，生成开发周报：完成功能、修复 bug、下周计划"

6. 安全加固

已实施的安全措施

  • ✅ API Key 权限最小化（DeepSeek Key 仅限 API 调用）
  • ✅ Gateway allowlist（只允许指定用户/群）
  • ✅ 敏感文件权限设置（.env 仅当前用户可读）
  • ✅ Tirith 安全扫描已配置
  • ✅ 命令执行确认（高危操作需二次确认）

每月安全检查

  1. 检查 API 用量：登录 DeepSeek 后台 → 查看本月消耗
  2. 检查 Gateway 日志：hermes gateway logs --since "30 days ago" | grep ERROR
  3. 更新 Hermes：hermes update

🚨 API Key 泄露应急流程

  1. 立即登录 DeepSeek 后台 → 删除泄露的 Key
  2. 生成新 Key
  3. 更新 ~/.hermes/.env
  4. 重启 Hermes
  5. 联系我确认安全

👤 团队成员离职流程

  1. 从 Gateway allowlist 移除该成员
  2. 如果该成员接触过 API Key，轮换所有 Key
  3. 检查最近的执行日志是否有异常操作

7. 团队使用指南

角色与权限

角色能做什么不能做什么
管理员（你）全部操作、修改配置、管理成员-
开发者通过平台 @Agent 提问、使用定时任务输出修改 Agent 配置
查看者接收定时任务推送直接与 Agent 交互

团队接入步骤

  1. 成员在对应平台加入团队群
  2. 管理员将成员 ID 加入 allowlist
  3. 成员发送 @AgentX 你好 测试
  4. 收到回复即接入成功

提问最佳实践

  • 具体比模糊好：「查上周销售数据」>「查数据」
  • 给上下文：「Next.js 14，App Router 模式，想加一个…」
  • 追问别怕：「再详细点」「给我代码示例」
复杂任务让一个人统一与 Agent 对话，避免多人同时下指令造成混淆。

8. 运维与监控

每周检查（手动，5 分钟）

hermes cronjob list              # 确认任务状态正常
hermes gateway logs --since "7 days ago" | tail -20  # 扫一眼异常

每月检查（手动，10 分钟）

  1. 登录 DeepSeek 后台查用量
  2. 检查磁盘空间（对话日志会累积）
  3. 备份自定义技能文件
  4. 评估是否需要调整定时任务

备份命令

tar -czf agentx-backup-$(date +%Y%m%d).tar.gz \
  ~/.hermes/skills/custom/ \
  ~/.hermes/.env \
  ~/.hermes/config.yaml \
  ~/.hermes/USER.md \
  ~/.claude/CLAUDE.md

升级流程

  1. 备份当前配置（见上面）
  2. 更新 Hermes：hermes update
  3. 更新 Claude Code：claude --update
  4. 测试：各发一条消息确认正常
