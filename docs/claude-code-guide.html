Claude Code 安装指南

覆盖 macOS / Windows / Linux 三大平台
装好就能在终端里让 Claude 读代码、改代码、提交 PR

🍎 macOS 🪟 Windows 🐧 Linux
Before You Start

前置准备

📝
Claude Code 的安装非常简单——不需要装 Node.js、Python 或任何运行时。官方原生安装器会处理一切，只需一条命令。
1

准备好 Anthropic 账号

你需要以下 二选一：

方案 A：订阅账号方案 B：API Key
来源claude.ai 注册console.anthropic.com 获取
费用$20/月（Pro）起按量付费，用多少扣多少
适合个人日常使用，交互式开发自动化、CI/CD、用量可控
认证方式浏览器 OAuth 登录设置环境变量
💡
推荐新手选方案 A（订阅 Pro），$20/月含 Claude Code 使用权，省去 API 配置的麻烦。进阶用户选方案 B（API Key），按量计费更灵活。
2

确认 Git 已安装

终端里运行 git --version，显示版本号就 OK。没有的话去 git-scm.com 下载安装。

3

国内用户必读：网络方案

Claude Code 需要访问 api.anthropic.com，国内可能无法直连。你有三个选择：

方案操作难度
🟢 代理/VPN开全局代理，终端设 https_proxy简单
🟡 API 中转用中转服务（需自行寻找可靠提供商）中等
🔵 DeepSeek 适配通过 ANTHROPIC_BASE_URL 指向 DeepSeek 的 Anthropic 兼容接口中等

具体配置方法见下方 「国内用户配置」 章节。

Choose Your Platform

安装 Claude Code

各平台都只需一条命令，选你的系统开始

1

运行安装命令

打开终端（Terminal.app），粘贴这条命令，回车：

bash 
curl -fsSL https://claude.ai/install.sh | bash

安装程序会自动下载并配置好一切。几十秒就完成。

🍺
如果你有 Homebrew，也可以：brew install --cask claude-code
2

刷新终端

让终端识别新的 claude 命令：

bash 
source ~/.zshrc

或者直接关掉终端重新打开。

3

验证安装

bash 
claude --version

显示版本号（如 1.0.61）就成功了。再运行诊断确认一切正常：

bash 
claude doctor
⚠️
Claude Code 不支持 Windows 原生运行，必须使用 WSL2。
WSL2 是 Windows 自带的 Linux 子系统，安装一次即可，之后就跟用 Linux 一样。
1

安装 WSL2（如已安装跳过）

以管理员身份打开 PowerShell，运行：

powershell（管理员） 
wsl --install -d Ubuntu

安装完成后重启电脑。重启后会自动打开 Ubuntu 终端，设置用户名和密码。

2

配置 WSL2 网络

在 Windows 上创建 %USERPROFILE%\.wslconfig 文件，写入：

.wslconfig 
[wsl2]
networkingMode=mirrored

然后 PowerShell 里运行 wsl --shutdown，等几秒后重新打开 Ubuntu。

3

在 WSL2 里安装 Claude Code

打开 Ubuntu 终端，运行：

bash（WSL2 内） 
curl -fsSL https://claude.ai/install.sh | bash
bash 
source ~/.bashrc
4

验证

bash 
claude --version
claude doctor
1

安装基础依赖

大多数发行版已自带。如果没有 curl：

bash 
# Debian / Ubuntu
sudo apt update && sudo apt install curl -y

# Fedora
sudo dnf install curl -y
2

运行安装命令

bash 
curl -fsSL https://claude.ai/install.sh | bash
3

刷新终端

bash 
source ~/.bashrc
4

验证

bash 
claude --version
claude doctor

如果提示 command not found：

bash 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
Authentication

登录与认证

安装完成后，你需要登录才能使用 Claude Code

1

首次启动

在终端里输入：

bash 
claude

第一次运行会自动打开浏览器，让你登录 Anthropic 账号（Pro/Max 订阅用户）或输入 API Key。

🔑
用 API Key？ 提前在终端设置环境变量：export ANTHROPIC_API_KEY="sk-ant-..."，然后运行 claude。建议把这行加到 ~/.zshrc 里持久化。
2

确认登录状态

进入 Claude Code 后，输入：

claude 
/status

会显示你的账号类型、当前模型、使用量等信息。看到绿色的连接状态就 OK。

Configuration

配置 CLAUDE.md

这是 Claude Code 最重要的配置文件——告诉它你的项目怎么跑

💡
CLAUDE.md 是什么？ 一份项目说明书。Claude Code 每次打开项目都会先读它，所以不用每次都解释"我这个项目是干嘛的"。它会跟着 Git 提交，全团队共享。
1

最快的生成方式：让 Claude 自己写

进入你的项目目录，启动 Claude Code，然后输入：

claude 
/init

Claude 会自动扫描你的代码库（package.json、目录结构、配置文件...），生成一份定制的 CLAUDE.md。你只需要检查一下，微调几个地方就行。

2

CLAUE.md 写什么？（参考模板）

不需要写长篇大论，就告诉 Claude 这几件事：

markdown CLAUDE.md 
# 项目：XXX

## 常用命令
- `npm run dev` — 启动开发服务器
- `npm test` — 运行测试

## 代码风格
- 用 TypeScript 严格模式
- 组件用函数式写法

## 规则
- 永远不要提交 .env 文件
- PR 前先跑 lint

就这些就够了。以后用着用着慢慢加，不用一开始就写得很完美。

3

测试 CLAUDE.md 是否生效

在 Claude Code 里问一个关于你项目的问题，比如：

claude 
这个项目的开发服务器怎么启动？

如果它能直接回答出来（而不是反问你），说明 CLAUDE.md 配好了。

China Users

国内用户配置

在国内网络环境下使用 Claude Code 的三种方案

A

代理/VPN（最简单）

如果你有代理软件（Clash、Surge 等），在终端里设置代理环境变量：

bash 
# 添加到 ~/.zshrc 或 ~/.bashrc（端口改成你代理的端口）
export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890

保存后 source ~/.zshrc 生效。验证：curl -I https://api.anthropic.com 应该返回 200。

B

API 中转服务

将 Claude Code 指向一个国内可访问的中转 API：

bash 
export ANTHROPIC_BASE_URL="https://your-proxy.com"
export ANTHROPIC_AUTH_TOKEN="sk-your-key"
⚠️
中转服务需自行寻找可靠提供商。建议先小额充值测试稳定性和响应速度，确认支持提示缓存（能省约 70% 费用）后再正式使用。
C

接入 DeepSeek（性价比最高）

DeepSeek 提供了 Anthropic 兼容接口，可以代替 Claude：

bash 
export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
export ANTHROPIC_API_KEY="sk-your-deepseek-key"
export ANTHROPIC_MODEL=deepseek-chat
export ANTHROPIC_SMALL_FAST_MODEL=deepseek-chat
💰
DeepSeek 价格约为 Claude 的 1/10，而且国内直连无需代理。缺点是代码理解和复杂任务能力弱于 Claude 原生模型。适合预算敏感或日常简单任务。

Troubleshooting

常见问题

claude: command not found？▾
终端没有刷新 PATH。运行 source ~/.zshrc（macOS）或 source ~/.bashrc（Linux/WSL），或者关掉终端重开。
如果还不行：echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
提示 "Auth conflict" 冲突？▾
你同时设置了 OAuth 登录和 API Key。二选一即可：
• 用订阅：在 Claude Code 里输入 /logout 登出 API Key
• 用 API Key：取消设置环境变量 unset ANTHROPIC_API_KEY，然后用 /logout 登出账号
Windows 装不了？▾
Claude Code 不支持 Windows 原生。必须装 WSL2：PowerShell（管理员）里运行 wsl --install -d Ubuntu，重启后在 Ubuntu 里按 Linux 流程安装。
国内连不上 api.anthropic.com？▾
三种方案（详见上方「国内用户配置」）：
1. 开代理（最简单）
2. 用 API 中转服务
3. 接入 DeepSeek 兼容接口（最便宜）
Claude Code 和 Claude.ai 网页版有什么区别？▾
Claude Code 是终端里的 AI 编程助手——能直接读你的代码、编辑文件、运行命令、提交 Git。Claude.ai 是网页聊天，只能粘贴代码让它看。如果你做开发，Claude Code 体验强很多。
后续使用需要付费吗？▾
需要 Anthropic 的付费账号或 API 额度：
• Pro 订阅：$20/月，含 Claude Code 使用权（有使用量限制）
• API Key：按量付费，用多少扣多少，适合高频使用
• DeepSeek 适配：走 DeepSeek 的价格（约 ¥1-3/百万 token），便宜很多
