Before You Start

前置准备

📝
Codex 基于 Node.js —— 需要先装好 Node.js 18+。安装包自带 npm，一条命令就能搞定 Codex。
1

装好 Node.js 18+

终端里跑一下，确认版本 ≥ 18：

bash 
node --version
npm --version

没装的话去 nodejs.org 下载 LTS 版本安装，或者用 Homebrew：brew install node。

2

准备好 API Key

Codex 需要大模型 API Key。你有两个选择，可以只用一个，也可以两个都配：

方案 A：OpenAI API方案 B：DeepSeek API 🔥
获取platform.openai.complatform.deepseek.com
费用按量付费，GPT-5 较贵极便宜，约 ¥1-2/百万 token
网络需要代理/VPN国内直连，无需代理
适合最强模型、复杂任务日常使用、性价比首选
💡
推荐两个都配。 日常用 DeepSeek（便宜直连），复杂任务切到 OpenAI 模型。Codex 支持在对话中随时切换模型。
3

网络说明

如果你用 DeepSeek：不需要任何代理，国内网络直连。

如果你用 OpenAI：需要能访问 api.openai.com。开全局代理，或在终端设：

bash 
export https_proxy=http://127.0.0.1:7890

安装 Codex

全局装一次，所有项目都能用

1

全局安装 Codex

打开终端（Terminal.app），一条命令：

bash 
npm install -g @openai/codex

几十秒装完。npm 会自动把 codex 命令加到系统路径。

2

验证安装

bash 
codex --version

显示版本号就成功了。

如果提示 command not found，关掉终端重新打开，或者手动刷新：

bash 
source ~/.zshrc
💡
Windows 上推荐用 PowerShell 或 WSL2。如果用 WSL2，装完 Node.js 后跟 Linux 步骤一样。
1

装 Node.js（如已有跳过）

去 nodejs.org 下载 Windows 安装包（.msi），一路下一步即可。

装完后打开 PowerShell，确认：

powershell 
node --version
npm --version
2

全局安装 Codex

在 PowerShell 或 WSL2 终端里运行：

powershell / bash 
npm install -g @openai/codex
3

验证

bash 
codex --version
1

装 Node.js 18+

推荐用 NodeSource：

bash 
# Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Fedora
sudo dnf install nodejs -y

全局安装 Codex

bash 
npm install -g @openai/codex
3

验证

bash

如果 command not found：

bash 
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.bashrc

Configuration

配置 API Key 与模型

配置一次，全局生效

1

设置 API Key（推荐两个都配）

把下面两行加到 ~/.zshrc 或 ~/.bashrc（按你用的 shell 选），然后 source 一下：

bash ~/.zshrc 
# OpenAI（需要代理）
export OPENAI_API_KEY="sk-your-openai-key"

# DeepSeek（推荐，国内直连，便宜）
export DEEPSEEK_API_KEY="sk-your-deepseek-key"
🔑
DeepSeek API Key 在 platform.deepseek.com 免费注册就能拿到。新用户一般有免费额度。
2

配置使用 DeepSeek 模型

在项目根目录创建 .codex.toml（或全局 ~/.codex/config.toml）：

toml .codex.toml 
# 默认使用 DeepSeek（便宜、国内直连）
[model]
provider = "deepseek"
model = "deepseek-chat"

# 可选：也保留 OpenAI，需要时切过去
[[model.aliases]]
provider = "openai"
model = "gpt-5"
alias = "gpt5"

配置好后启动 Codex，它会自动用 DeepSeek。想临时切到 OpenAI 模型，在对话里直接说「用 GPT-5 来干这个」就行。

3

启动 Codex

进入你的项目目录，运行：

bash 
codex

首次启动会验证 API Key。看到对话界面就配置成功了。

想确认当前用的什么模型，在 Codex 里输入：

codex 
/status
Project Setup

配置项目说明书

让 Codex 一进项目就知道你是谁、要干嘛

💡
Codex 跟 Claude Code 一样有项目记忆。 在项目根目录放一个 CODEX.md（或 .codex/instructions.md），Codex 每次打开项目都会先读它。
1

生成项目说明书

让 Codex 自己写——在项目目录里启动 Codex，输入：

codex 
/init

Codex 会扫描你的项目结构，自动生成一份 CODEX.md。你检查一下就行。

2

CODEX.md 模板参考

不需要长篇大论，告诉 Codex 这四件事：

markdown CODEX.md 
# 项目：XXX

## 常用命令
- `npm run dev` — 启动开发服务器
- `npm test` — 运行测试

## 代码风格
- 用 TypeScript 严格模式
- 组件用函数式写法

## 规则
- 永远不要提交 .env 文件
- PR 前先跑 lint
Pro Tips

DeepSeek 接入进阶

几个你可能用得到的配置技巧

1

同一个 Codex，多个模型随时切换

不需要改配置文件。在 Codex 对话里直接用自然语言说：

  • 「用 deepseek 来写这段代码」→ 切到 DeepSeek
  • 「换 GPT-5，这个 bug 太复杂了」→ 切到 OpenAI
  • 「用 V4 模型跑一下」→ 切到 DeepSeek V4

Codex 会自动切换对应的 API。

2

DeepSeek 免费额度用完后

DeepSeek 充值极便宜，¥1 能用很久。在 platform.deepseek.com 充值，最低 ¥10 起。日常写代码一个月也就几块钱。

3

团队共享配置

把 .codex.toml 放到项目根目录，提交到 Git。团队其他人 clone 下来就能直接用的 DeepSeek 配置，不需要每个人自己折腾。

⚠️
别把 API Key 写进 .codex.toml 然后提交！ Key 用环境变量，配置文件只写 provider 和 model。把 .codex.toml 提交到 Git、.env 加到 .gitignore。
